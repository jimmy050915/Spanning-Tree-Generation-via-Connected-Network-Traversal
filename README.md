# 小说人物关联分析系统

当前仓库已完成开发文档中的第一阶段“领域数据结构”。正式实现位于根目录的 `src/domain`，`legacy/` 仅保留为早期邻接多重表原型，不参与正式构建。

## 阶段一内容

- 使用稳定 `PersonId` 的人物顶点。
- 一条无向关系只创建一个 `EdgeNode` 的邻接多重表。
- 人物增删、重命名与章节计数维护。
- 关系边插入、删除、查找和稳定排序枚举。
- 根据人物出现章节数和共同章节数计算 Jaccard 关联度。
- 带错误码和中文诊断信息的完整图结构验证。
- 孤立人物、非连通图和桥边删除支持。

章节导入、人物别名、图重建、DFS/BFS、二进制持久化和 Qt 图形界面属于后续阶段，本阶段没有提前实现。

## 构建与测试

项目要求 CMake 3.21 或更高版本及支持 C++17 的编译器。在 Windows PowerShell 中执行：

```powershell
.\scripts\verify.ps1
```

脚本会配置、构建并运行全部 CTest。由于当前工程路径包含中文，脚本会临时选择未占用的 `R:`、`S:`、`T:` 或 `U:` 盘符映射工程目录，并在结束时自动解除；构建产物保存在根目录对应的 `build-mingw-r`、`build-mingw-s` 等目录，且已被 Git 忽略。按盘符隔离缓存可避免某个盘符被占用后 CMake 读取到其他映射留下的缓存。

在纯英文路径中也可以直接执行：

```powershell
cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug
cmake --build build
ctest --test-dir build --output-on-failure
```

## 领域层约束

人物和边由 `std::unique_ptr` 所在容器统一拥有，边链指针均为非拥有指针。公开修改操作会在修改前后验证图状态；失败的业务请求不会改变已有图。正式代码不包含可修改的业务全局变量，也不使用固定长度数组保存人物或关系。
