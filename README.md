# 小说人物关联分析系统

当前仓库已完成开发文档中的第一阶段“领域数据结构”、第二阶段“章节与统计”、第三阶段“图算法”和第四阶段“文件系统”。正式实现位于根目录的 `src/domain` 与 `src/infrastructure`，`legacy/` 仅保留为早期邻接多重表原型，不参与正式构建。

## 阶段一内容

- 使用稳定 `PersonId` 的人物顶点。
- 一条无向关系只创建一个 `EdgeNode` 的邻接多重表。
- 人物增删、重命名与章节计数维护。
- 关系边插入、删除、查找和稳定排序枚举。
- 根据人物出现章节数和共同章节数计算 Jaccard 关联度。
- 带错误码和中文诊断信息的完整图结构验证。
- 孤立人物、非连通图和桥边删除支持。

## 阶段二内容

- 带稳定 `ChapterId` 的章节记录、唯一章节键和章内人物去重。
- 复用图名称索引的标准人物词典，以及直接映射到 `PersonId` 的别名字典。
- 由 `NovelRelationProject` 统一管理人物、别名、章节和派生关系图。
- 以全部章节为权威来源计算人物出现次数、人物对共现次数和 Jaccard 关联度。
- 章节增改删使用候选章节集合和原子图统计替换，失败时保持原项目不变。
- 全量重建保留人物对象和仍存在关系的 `EdgeId`，已删除的边编号不复用；重建会替换边对象，调用方不得继续使用重建前取得的 `EdgeNode` 指针或引用。
- 项目级验证会从章节重新计算统计，并检查章节、别名和人物关系图的一致性。

## 阶段三内容

- 只读 `IGraphView` 隔离遍历算法与邻接多重表内部存储，并保证顶点和邻居按人物编号稳定枚举。
- DFS 使用显式栈模拟递归式深度优先过程，BFS 使用队列；两者都按低编号邻居优先访问。
- 支持仅遍历起点所在连通分量，以及默认覆盖全部非连通分量的完整遍历。
- 完整遍历树使用 `PersonId{0}` 作为“全部人物”虚拟根，各连通分量起点按稳定顺序挂接；访问序列只包含真实人物。
- 使用孩子链表保存首次发现关系，支持按发现顺序读取孩子，并拒绝重复结点、自环、重复父子关系、多父结点和成环关系。
- 遍历结果同时返回稳定访问序列和结构化遍历树，供后续应用服务与界面层使用。

## 阶段四内容

- `Utf8TextFileLoader` 以二进制模式读取文本，默认限制为 20 MiB；支持文件头 BOM，并严格拒绝截断、过长编码、代理项和超出 Unicode 范围的非法 UTF-8。
- `ChapterTextParser` 支持 LF、CRLF 和 CR 换行，解析文件开头的 `@chapter=`、`@title=` 与正文；缺少章节编号时保留空值供后续界面补录，重复或正文后的元数据会返回带行号的错误。
- 人物词典解析会忽略空行和 `#` 注释并拒绝重复名称；别名词典要求一个 TAB 分隔符，并验证目标标准名、别名唯一性和名称冲突。
- `DictionaryNameExtractor` 对标准名与别名建立不可变词典快照，按较长名称优先从左到右匹配，输出排序、去重后的稳定 `PersonId`。
- `.nprg` 项目文件使用固定 `NPRGRAPH` 文件头、显式 little-endian 整数、带版本和长度的区段，以及覆盖完整数据区的 32 字节 SHA-256 摘要；未知区段可按长度安全跳过。
- 项目元数据区保存 `nextPersonId`、`nextEdgeId` 和 `nextChapterId`。加载先恢复到无裸指针的 `ProjectSnapshot`，再重建人物索引、章节索引和邻接多重表边链，因此删除产生的编号空洞和高水位不会丢失或复用。
- 保存先验证当前项目，在目标目录写入唯一临时文件、刷新到磁盘并重新加载验证；Windows 使用 `ReplaceFileW`/`MoveFileExW` 完成原子提交，失败时旧项目文件保持可用。
- 加载始终返回通过摘要、格式、图结构及章节统计完整验证的新候选项目，不会边读取边修改当前项目。

Qt 图形界面属于第五阶段，当前没有提前实现。

## 构建与测试

项目要求 CMake 3.21 或更高版本及支持 C++17 的编译器。在 Windows PowerShell 中执行：

```powershell
.\scripts\verify.ps1
```

脚本会配置、构建并运行全部 CTest。由于当前工程路径包含中文，脚本会临时选择未占用的 `R:`、`S:`、`T:` 或 `U:` 盘符映射工程目录，并在结束时自动解除；构建产物保存在根目录对应的 `build-mingw-r`、`build-mingw-s` 等目录，且已被 Git 忽略。按盘符隔离缓存可避免某个盘符被占用后 CMake 读取到其他映射留下的缓存。

在纯英文路径中也可以直接执行：

```powershell
cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug
cmake --build build
ctest --test-dir build --output-on-failure
```

## 领域层约束

人物和边由 `std::unique_ptr` 所在容器统一拥有，边链指针均为非拥有指针。项目聚合只公开子对象的只读视图，正式章节统计只能从章节记录重建。公开修改操作保持强异常安全；失败的业务请求不会改变已有章节、人物关系图或编号状态。文件读取只生成文本 DTO，持久化只处理字节与 `ProjectSnapshot`，二者都不直接修改邻接多重表内部结构。正式代码不包含可修改的业务全局变量，也不使用固定长度数组保存人物、关系或章节。
